## Checkout Repository

```bash
# HTTP
git clone https://github.com/anthony-mariotti/whendarr.git

# SSH (Recommended)
git clone git@github.com:anthony-mariotti/whendarr.git
```

## Using VSCode Workspace

It's recommend to use the provided
[VSCode workspace](https://code.visualstudio.com/docs/editing/workspaces/workspaces)
when developing Whendarr. We use Multi-root workspaces that allows you to configure
multiple distinct folders to be part of the same workspace. Instead of opening a folder
as workspace, you open `whendarr.code-workspace` JSON file that lists all folders of the workspace.

## Install Project Dependencies

```bash
cd /path/to/whendarr

pnpm install
```

## Running Locally

Development uses [Valkey](https://valkey.io/) instead of [Redis](https://redis.io/) to ensure compatability. Since Whendarr
relies on [ioredis](https://ioredis.com/), we want to catch any differences early.

```bash
# Spin up local valkey container
pnpm docker:valkey:up
```

Setup environment variables by copying the `.env.example`

```bash
cp .env.example .env
```

Key environment variables

- SONARR_URL
- SONARR_KEY
- RADARR_URL
- RADARR_KEY

> NOTE: Working on getting dynamic sample data so sonarr/radarr instances are not required to run durning development. At this time,
> a working sonarr and radarr instance is **required**.
